# Interativa SGP Notifier

Projeto desenvolvido a partir de uma demanda real da empresa em que trabalho, com o objetivo de integrar o **SGP** ao **Telegram** para enviar ao grupo dos técnicos as **ocorrências em aberto do dia**. A ideia surgiu da necessidade de automatizar uma tarefa operacional importante: dar visibilidade rápida para a equipe técnica sobre as ocorrências abertas, sem depender de consultas manuais no sistema logo no início do expediente.

## Contexto do projeto

Fui acionado pela empresa onde trabalho para criar uma integração que consumisse dados do SGP e entregasse essas informações de forma organizada em um grupo do Telegram da equipe técnica.

Esse projeto foi muito importante para mim porque me permitiu **consolidar e aprofundar conhecimentos fundamentais no desenvolvimento backend**, especialmente em pontos como:

- lógica de programação
- consumo de API externa
- organização de projeto em camadas
- tratamento de dados
- automação de tarefas
- integração entre serviços

Além disso, foi uma excelente oportunidade para **fortalecer minha visão de estrutura backend** e ampliar meu domínio sobre bibliotecas e ferramentas importantes no processo, como o **APScheduler**, além de compreender com mais profundidade a comunicação entre o nosso sistema e servidores externos, como o do SGP e o do Telegram.

## Objetivo

Automatizar o envio de ocorrências em aberto para o grupo técnico, melhorando a visibilidade operacional da equipe e reduzindo o trabalho manual de consulta no sistema.
